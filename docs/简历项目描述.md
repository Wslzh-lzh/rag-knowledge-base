# RAG 企业知识库问答系统

## 项目描述

基于 RAG（检索增强生成）技术打造的企业知识库问答系统，支持 PDF、Markdown、HTML、TXT 等多格式文档上传，通过向量检索 + 全文检索 + Rerank 重排的混合检索方式，结合大语言模型实现流式问答和来源追溯。系统采用前后端分离架构，支持 Docker 一键部署，包含用户认证、知识库管理、文档管理、对话历史等完整功能。

---

## 技术栈

**后端：** Python 3.12、FastAPI、SQLAlchemy、PostgreSQL、SQLite、Qdrant、Redis、OpenSearch、PyMuPDF、JWT

**前端：** Next.js 14、React 18、TypeScript、Tailwind CSS

**其他：** Docker、Alembic、MinIO

---

## 主要职责与成果

1. 负责系统整体架构设计，采用 Provider 模式将 LLM、Embedding、向量数据库、全文搜索等核心组件抽象为可插拔接口，方便本地开发与生产环境灵活切换

2. 实现文档解析 pipeline，支持 TXT、Markdown、HTML、PDF 四种格式；针对 PDF 做了专门优化，使用 PyMuPDF + pypdf 双解析器保证兼容性，实现了带页码的语义分块，回答中的引用可以精确到具体页码

3. 搭建混合检索引擎，结合向量检索的语义理解能力和全文检索的关键词匹配优势，通过 RRF 算法融合两路结果，再经过 Reranker 二次排序，显著提升了检索准确率

4. 开发 RAG 问答编排模块，串联检索、重排、上下文构建、LLM 生成、SSE 流式输出完整链路；支持多轮对话，回答会标注每段内容的来源文档和页码，点击可跳转查看原文

5. 负责前端核心页面开发，包括知识库列表、文档管理、聊天问答、检索页面等；实现了 Markdown 格式回答渲染和代码块高亮，支持一键复制回答和代码；添加了文件拖拽上传功能，优化了多处交互体验细节

6. 搭建异步任务处理机制，超过 10MB 的文件和 PDF 自动走后台任务队列，上传接口立即返回，避免 HTTP 超时；元数据在向量化前先行落库，确保即使嵌入失败也不会丢失已解析的内容

7. 关注系统稳定性和可运维性，实现了 API 限流、安全响应头、GZip 压缩、请求 ID 追踪、访问日志等中间件；设计了优雅降级机制，OpenSearch 不可用时自动切到 PostgreSQL 全文检索，不影响主流程使用

8. 做了多处性能优化：DashScope Embedding 批量调用分批处理（每批不超过10条）、Qdrant 客户端单例懒加载、数据库连接池调优等；提供 Docker Compose 部署方案和健康检查接口，方便运维部署
