# RAG 知识库项目描述

## 项目简介

基于 RAG（检索增强生成）技术实现的知识库问答系统，支持多格式文档上传、解析、混合检索、流式问答、引用来源展示和对话管理。项目采用前后端分离架构，适合企业知识问答、资料检索和培训内容辅助查询等场景。

## 技术栈

### 后端

- Python
- FastAPI
- SQLAlchemy
- SQLite / PostgreSQL
- Qdrant / 内存向量存储
- OpenSearch / PostgreSQL 全文检索
- PyMuPDF / pypdf

### 前端

- Next.js 14
- React 18
- TypeScript
- Tailwind CSS

### 其他

- Docker Compose
- JWT 认证
- SSE 流式输出

## 项目亮点

### 1. 设计可插拔的 Provider 架构

将 Embedding、LLM、Reranker、Vector Store、Full-Text Search 等核心能力抽象为可切换 Provider，支持：

- 本地开发模式下使用 `mock / echo / memory`
- 生产环境接入真实外部服务
- 降低不同模型与存储实现的耦合成本

### 2. 打通完整 RAG 主链路

完成从：

`文档上传 -> 文档解析 -> 文本分块 -> 检索召回 -> 重排 -> 上下文构建 -> 流式回答 -> 引用来源展示`

的完整链路开发。

### 3. 支持多格式文档处理

支持：

- TXT
- Markdown
- HTML
- PDF

其中 PDF 支持页码感知，回答引用可精确到页码范围。

### 4. 实现混合检索能力

结合：

- 向量检索
- 全文检索
- Rerank 重排

提升检索命中质量，并支持按知识库过滤搜索范围。

### 5. 优化本地运行体验

针对本地开发场景做了多项调整：

- 默认数据库切换为 SQLite
- 默认管理员自动初始化
- 在无真实模型密钥时仍可通过 `echo / mock` 跑通主流程
- 修复后端启动时数据库初始化和部分测试稳定性问题

### 6. 完成知识库与文档管理页面

前端实现了：

- 登录页
- 首页
- 知识库列表页
- 知识库详情页
- 搜索页
- 聊天页

支持文档上传、预览、重命名、删除、重新解析、在线编辑等能力。

### 7. 改善聊天体验

在聊天页面中实现了：

- 流式问答展示
- Markdown 回答渲染
- 代码块复制
- 回答复制
- 引用来源折叠展开，避免长来源列表撑高消息卡片

## 可用于简历的表述

负责开发企业级 RAG 知识库问答系统，基于 FastAPI + Next.js 搭建前后端分离架构，实现文档上传解析、混合检索、流式问答、引用来源展示与对话管理；设计 Provider 可插拔架构，支持本地零依赖模式和多模型/多存储后端切换。

## 可用于面试的重点话题

这个项目适合延展开聊的方向包括：

- 什么是 RAG，和纯大模型问答的区别
- 混合检索为什么比单一路径更稳
- Provider 模式如何提升系统可扩展性
- PDF 页码感知分块如何服务于引用来源展示
- SSE 流式输出在前端如何消费
- 为什么本地开发要支持 mock / echo / memory 模式
